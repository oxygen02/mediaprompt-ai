## 一、产品概述

### 1.1 产品定位
MediaPrompt AI 是一款面向非技术用户的内容反向工程工具，帮助用户从已有的优质内容中提炼有效的 AI 提示词，实现"让美好可以复刻迭代"。

**核心价值主张：最好的学习从模仿开始**

### 1.2 目标用户
- **主要用户**：不懂提示词的小白用户（内容创作者、运营人员、设计师）
- **次要用户**：需要快速生成提示词的专业用户

### 1.3 核心价值
- **学习效率**：从模仿开始，快速掌握优质内容的创作逻辑
- **降低门槛**：无需深入学习提示词工程，一键生成专业提示词
- **可复用性**：生成的提示词可用于多种 AI 模型和工具

### 3.2 设计规范

#### 颜色方案
- **背景色**：`#f5f5f7`（浅灰）
- **侧边栏背景**：`#ffffff`（白色）
- **主色调**：`#52525b` ~ `#3f3f46`（黑灰色系）
- **边框色**：`#e5e5e5`（浅灰边框）

#### 月食动画（核心视觉元素）
- **顶部月食**：32px，旋转动画（20s周期）+ 吞噬动画（8s周期）
- **分类月食图标**：16px，不同黑化程度
  - 文档：90%黑（`moon-95`，阴影内嵌 14px）
  - 图片：70%黑（`moon-70`，阴影内嵌 11px）
  - 视频：40%黑（`moon-40`，阴影内嵌 6px）
  - 网页：10%黑（`moon-full`，阴影内嵌 2px）

#### 按钮样式
- **主按钮**：渐变背景（`#52525b` → `#3f3f46` → `#52525b`）+ 流动动画
- **加载状态**：脉冲动画 + "正在努力生成..."

#### 交互效果
- **导航项**：hover 时背景变浅，active 时背景加深
- **选项标签**：选中时显示实心圆点 + 边框加深
- **文件拖拽区**：hover 时边框颜色加深

| Profile | 详细描述AI助手的专业背景 | 深灰左边框 |

| Constrains | 列出约束条件 | 深灰左边框 |
| OutputFormat | 定义输出格式要求 | 深灰左边框 |
| Workflow | 列出工作流程步骤 | 中灰左边框 |

| Initialization | 初始化问候语 | 深灰左边框 |

## 五、用户故事

### 5.1 文档分析场景

**用户故事 1：短视频运营学习**
> 作为一名短视频运营新手，我想分析热门短视频的运营策略文档，以便学习专业运营的思路和方法。

**验收标准：**
- 用户可以上传 Word/PDF 格式的运营文档
- 系统生成结构化的运营策略提示词
- 提示词包含 Role、Background、Skills 等完整字段
- 用户可以复制提示词用于 ChatGPT/Claude 等工具

### 5.2 图片分析场景

**用户故事 2：AI 绘画提示词生成**
> 作为一名 AI 绘画爱好者，我想上传喜欢的图片，获取可用于 Midjourney/SD 的提示词。

**验收标准：**
- 支持 JPG/PNG/WebP/GIF 格式
- 系统分析图片的主体、风格、构图、光影、色彩
- 输出 Midjourney/SD/DALL-E 兼容的提示词
- 提供参数建议（分辨率、采样器等）

## 六、技术架构

### 6.1 技术栈

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| 前端 | Next.js 16 + Tailwind CSS 4 + TypeScript | React 框架 |
| 后端 | Node.js + Express | API 服务 |
| AI | 腾讯云混元大模型 | 文本/图片分析 |
| 存储 | 腾讯云 COS | 静态资源托管 |

### 6.2 API 接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/api/analyze/text` | POST | 文本分析 |
| `/api/analyze/image` | POST | 图片分析 |
| `/api/analyze/file` | POST | 文件分析 |
| `/api/preview` | POST | 模拟预览 |

## 七、定价策略

- 每日 10 次分析
